We have a Flask customer-support API with inconsistent authorization: some routes declare authentication decorators but the JWT middleware doesn't enforce them at runtime, and a few routes have no decorator at all yet the middleware enforces authentication anyway. The goal across three milestones is to write Rego policy modules that detect and report each class of inconsistency. You do not modify the Flask source.

For this first milestone, parse the Flask source at /app/api/routes.py and the route metadata at /app/route_metadata.json, then produce a consolidated policy document at /app/route_policies.json. Each entry in the output should cover one route and include the route path, HTTP method, the decorator applied to it in the Flask source (one of "require_auth", "require_admin", or "none"), the list of JWT claim names that decorator requires, sorted alphabetically (["role", "sub"] for "require_auth" and "require_admin" decorators; [] for public routes with decorator "none"), and whether the JWT middleware actually enforces authentication for that route according to the metadata.

The output must be a JSON array sorted alphabetically by route path. The field names are: route, method, decorator, required_claims, middleware_enforced.
