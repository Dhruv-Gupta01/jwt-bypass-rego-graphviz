With route_policies.json in place, write Rego rules in a file named findings.rego using `package jwt_bypass`. OPA v0.68.0 requires the `contains` keyword form for set rules so the output serializes as a JSON array. Define a set named `findings` that identifies six classes of JWT bypass. When loading data, combine route_policies.json (array, key `route_policies`), access_logs.ndjson parsed into an array (key `access_logs`), and route_calls.json (key `route_calls`) into one JSON object and pass it via `--data`. Your Rego rules must reference this data using the `data.*` namespace (e.g., `data.route_policies`, `data.access_logs`, `data.route_calls`) — not `input.*`. The route_calls.json is an object where each key is a route path and its value is an array of route paths that it calls internally. Each log entry has: timestamp, route, method, bearer_token, jwt_valid (bool), scope_checked (bool), claims (object with sub and role), status. Write the output to /app/findings.json as a JSON array sorted by [route, finding_type]. Each entry needs route, finding_type, and evidence fields. Each (route, finding_type) pair appears exactly once.

The six finding types are: DECORATOR_MIDDLEWARE_MISMATCH when a route's decorator and middleware_enforced disagree (sub-cases: auth decorator but middleware_enforced=false tagged "auth_not_enforced", or decorator=none but middleware_enforced=true tagged "shadow_enforcement"; evidence: decorator string and middleware_enforced bool). SCOPE_NOT_CHECKED when a properly-enforced route has log entries where jwt_valid=true but scope_checked=false (skip routes already flagged as DECORATOR_MIDDLEWARE_MISMATCH; evidence: jwt_valid true, scope_checked false, observed_count integer, affected_subs sorted array of unique claims.sub values). ROLE_NOT_VALIDATED when a require_admin route has log entries where jwt_valid=true, scope_checked=false, and claims.role != "admin" (evidence: non_admin_count integer, non_admin_subs sorted array of unique claims.sub). PRIVILEGED_SCOPE_BYPASS when a properly-enforced route has scope-bypassing log entries exclusively from admin tokens and no ROLE_NOT_VALIDATED applies (evidence: admin_bypass_count integer). A route can have both SCOPE_NOT_CHECKED and PRIVILEGED_SCOPE_BYPASS if the conditions for both are independently met. CASCADING_BYPASS when a SCOPE_NOT_CHECKED route is called directly by a DECORATOR_MIDDLEWARE_MISMATCH route (evidence: upstream_mismatch_routes sorted array, upstream_mismatch_types object mapping each upstream route to "auth_not_enforced" or "shadow_enforcement"). UNENFORCED_SENSITIVE_ROUTE when decorator=none and middleware_enforced=false and the path matches /admin/* or /internal/* (evidence: decorator "none", middleware_enforced false, path_pattern the matched glob).
